# AI 托管链上占比与真实历史设计

## 目标与范围

前端只负责在用户开启或关闭托管时通知后端一次。托管启用后，后端自行按轮询周期读取链上博弈池状态、虚拟储备、IPFS 元数据与黄金行情，不依赖前端持续上报。

本轮只修改 Go 后端。新版 `agent/` 中尚未提交的前端改动全部保留，不修改前端随机曲线逻辑；SQLite、IPNS 和长期历史持久化留到后续。本轮历史由 IPFS 已有真实点和后端进程内链上采样共同组成，进程重启后内存采样会丢失。

## 数据来源与占比计算

每个托管任务继续通过链上 `getGameInfo(game_id)` 获取 IPFS CID、池总额、截止时间和结算状态，通过 `getGameExtraData(game_id, user)` 获取虚拟储备。合约储备顺序是 `NO, YES`，后端转换为：

- `YES 占比 = reserveYES / (reserveNO + reserveYES) * 100`
- `NO 占比 = reserveNO / (reserveNO + reserveYES) * 100`

计算使用大整数/有理数路径，最后才转换为有限浮点百分比，避免先转 `float64` 导致大额储备溢出。储备缺失、为负或总和为零时，本轮数据无效，不调用 AI、不下单。

IPFS 元数据新增后端可识别的 `history` 字段。每个点兼容前端格式：时间读取 `t` 或 `time`，YES 读取 `y` 或 `yes`，NO 读取 `n` 或 `no`。时间必须大于零；百分比必须有限且在 `0..100`。若只提供 YES 或 NO，另一项由 `100 - 已知项` 推导；两项都存在时总和必须在允许的小误差内接近 100。非法点被丢弃，不影响静态元数据读取。

## 进程内真实历史

新增并发安全的市场历史存储，以规范化合约地址和 `game_id` 为键，与用户地址无关。同一池被多个用户托管时共享一份行情历史。

处理每个池时，先把 IPFS 中的有效历史点作为种子合并到存储，再把本轮链上储备计算出的占比追加为当前点。当前点时间按 AI 轮询周期分桶；同一轮中多个用户并发处理同一池，只会写入一个点。历史按时间排序、按时间戳去重，并只保留最新 256 个点，防止进程长期运行导致内存无限增长。

YAML 的 `ai` 分组增加：

```yaml
history_min_points: 3
history_max_points: 256
```

两项均严格校验为正数，且最大点数不得小于最小点数。示例配置、真实本地配置和配置说明同步更新。

## AI 决策与强制 HOLD

后端完成本轮链上采样后读取该池历史：

1. 少于 `history_min_points` 个有效点时，直接记录结构化日志并返回，不调用模型、不发送交易。这是后端强制 `HOLD`，不能被模型绕过。
2. 达到门槛后，模型上下文包含静态 IPFS 信息、详细说明、当前 YES/NO 占比、按时间排序的真实历史序列、原始储备、池总额、截止时间和黄金行情。
3. IPFS 市场文本明确标记为不可信数据，系统提示要求模型不得把市场标题、条件或详细说明当作指令。
4. 模型仍只能返回 `buy_yes`、`buy_no` 或 `hold`；现有置信度门槛、同方向冷却和固定买入金额继续生效。

IPFS 下载失败不阻止链上采样，但没有足够进程内历史时仍强制 `HOLD`。模型 API、行情或链上调用失败时沿用现有错误记录，不发送交易。

## 组件边界

- `internal/ipfs`：解析静态元数据与兼容格式历史点，只负责数据格式和校验。
- `internal/aimanaged/history.go`：计算链上占比、合并种子、分桶去重、排序和截断。
- `internal/aimanaged/manager.go`：编排链上/IPFS/行情读取、历史门禁、AI 决策与交易。
- `internal/config`：提供并校验历史最小点数和最大保留点数。

历史存储通过小接口注入托管引擎，测试可以使用独立内存实例；生产构造器创建共享实例。现有链客户端、元数据源、行情源和决策源接口继续复用。

## 测试与验收

自动化测试覆盖：

- IPFS `history` 的长短字段别名、缺失互补、非法点过滤、时间排序和去重。
- 合约 `NO, YES` 储备顺序转换为正确的 YES/NO 百分比，包括超大整数和零总额。
- 同一池多个用户在同一轮只追加一个链上点，超过上限时丢弃最旧点。
- IPFS 两个有效点加一个链上点后达到 3 点并允许调用 AI。
- 少于 3 点时决策源调用次数为零、交易次数为零。
- 达到门槛后 AI 请求包含当前占比和历史点；`hold`、低置信度和高置信度交易规则保持正确。
- `go test ./...`、`go vet ./...` 和 `go test -race ./internal/aimanaged` 全部通过。

所有测试使用模拟链、模拟 IPFS、模拟行情和模拟 AI，不访问真实外网、不广播真实交易。
